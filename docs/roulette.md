# Рулетка — устройство и обслуживание

Полностью автономная копия слот-машины из DBDStreaming.
**Без Flask, без SSE, без `/api/*`, вообще без сети** — серверная логика роллов
(`modules/rolls.py`, `modules/perk_randomizer.py`) портирована прямо в страницу,
все данные вшиты в `data/library.js`.

**Ничто не показывает, сколько спинов осталось** — счётчика нет ни в UI, ни в
памяти. Шансы и то, как собирается последовательность, намеренно не описаны
в документации: они живут в коде (`src/roulette_app.jsx`). Не переносите их
в README — он может оказаться на экране.

## Структура папки

| Путь | Что это |
|---|---|
| `index.html` | Страница. Открывать её. |
| `js/roulette_app.js` | Собранный бандл (React внутри). |
| `src/` | Исходники: `SlotMachine.jsx` (форк из основного приложения — см. ниже), `audio.js` (звук + мастер-громкость), `roulette_app.jsx` (панель + портированная логика роллов), `build.mjs` + `package.json` (пересборка). |
| `css/` | `slot_machine.css` (без изменений) + `app.css` (панель) + шрифты Cinzel/Orbitron. |
| `data/library.js` | **Генерируется** — пулы перков, библиотеки иконок, списки персонажей. |
| `data/*.json` | Исходные данные: `SurvivorPerks.json`, `KillerPerks.json`, `perk_map.json`, `survivor_map.json`, `killer_map.json`, `killer_perk_map.json`. |
| `tools/generate_library.py` | Перегенерирует `data/library.js` из JSON-ов и папок с картинками. |
| `survivorPerks/`, `killerPerks/` | Все иконки перков (включая варианты Teachables / With Backgrounds). |
| `survivorIcons/`, `killerIcons/` | Портреты персонажей для рулетки персонажей. |
| `sounds/slots/` | Все звуки слот-машины. |

## Чем `SlotMachine.jsx` отличается от копии в основном приложении

Начинался как точная копия; теперь это форк, потому что эта сборка управляется
вручную, а не дашбордом:

- **Никогда не прячется сам.** Оригинал держал результат 3 секунды, играл
  анимацию выхода и исчезал (таймингом владел дашборд). Здесь автомат остаётся.
- **Переключение полностью пересоздаёт автомат.** `roulette_app.jsx` передаёт
  меняющийся `key`, форсируя настоящий unmount. Рендер null (что делает
  `visible:false` сам по себе) сохраняет инстанс и его состояние — из-за этого
  золотое свечение переезжало в следующую рулетку, а звуки недокрученного спина
  продолжали играть. Не «упрощайте» этот key.
- **Рычаг — орган управления** (`onPull` / `canPull`), а не декорация.
- **Один триггер спина для обоих режимов** — каждый рывок инкрементит
  `spinIndex`. Оригинал автоматически крутил режим перков один раз при появлении.
- **Случайные стартовые позиции барабанов.** Оригинал всегда стартовал каждый
  барабан с индекса 0 — все четыре показывали одну и ту же иконку; никто этого
  не видел, потому что автомат появлялся уже в процессе ролла. Эта сборка стоит
  на холостом автомате между рывками, поэтому сеет случайные стартовые позиции.

Исправления рулетки в DBDStreaming сюда автоматически **не** попадают.

## После добавления/удаления картинок или правки JSON-ов

Перегенерируй вшитые данные (нужен Python 3):

```
python tools/generate_library.py
```

## Пересборка JS-бандла (только если правишь `src/*.jsx`)

Нужен Node:

```
cd src
npm install
npm run build
```
