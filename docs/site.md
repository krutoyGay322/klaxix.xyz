# Документация

Статический сайт: HTML + CSS + JavaScript. Без сборки, без зависимостей.
Заливается на GitHub Pages как есть — что в репозитории, то и на сайте.

## Структура

```
├── index.html          ГЛАВНАЯ — календарь ивентов
├── events.json         данные ивентов (заменяешь этот файл → календарь обновляется)
├── 404.html            страница «не найдено» в стиле сайта
├── .nojekyll           НЕ УДАЛЯТЬ (см. ниже)
├── .gitignore          что не попадает на GitHub
├── assets/             общие файлы главной страницы
│   ├── styles.css      дизайн: цвета, шрифты, компоненты
│   ├── event-types.js  типы ивентов (цвета/названия) — общие для календаря и редактора
│   └── calendar.js     логика календаря
├── docs/               документация (этот файл и инструкции)
├── editor/             РЕДАКТОР ивентов — ТОЛЬКО ЛОКАЛЬНО (в гите его нет)
├── CaptainShop/        МАГАЗИН КАПИТАНА → адрес сайта + /CaptainShop/
└── Roulette/           РУЛЕТКА → адрес сайта + /Roulette/
```

## Редактор ивентов

`editor/` в `.gitignore` — он не заливается на GitHub и на сайте недоступен.
Работает только с локального сервера: <http://localhost:8000/editor/>
(как запустить — ниже, «Локальный запуск»).

Из этого следует: папка живёт только на твоём диске. В свежем клоне
репозитория её не будет — если меняешь компьютер, скопируй `editor/` руками.

## Как добавить новый модуль (страницу)

1. Создай папку в корне (как `CaptainShop/`) со своим `index.html` внутри.
   Модуль самодостаточен: свои стили и скрипты держит у себя в папке.
2. В корневом `index.html` найди секцию `// ДРУГИЕ СТРАНИЦЫ` и скопируй
   карточку `<a class="pagecard" href="ПапкаМодуля/index.html">…` — поменяй
   название и ссылку.

   Ссылку пиши именно с `/index.html` на конце, а не просто `ПапкаМодуля/`.
   На сервере работают оба варианта, но если открыть сайт с диска (двойным
   кликом), ссылка на папку показывает список файлов вместо страницы.

## Локальный запуск

```bash
python -m http.server 8000
# открой http://localhost:8000
```

Открывать `index.html` двойным кликом нельзя — браузер заблокирует загрузку
`events.json`. Нужен сервер (команда выше). Сам магазин работает и с диска.

## Публикация на GitHub Pages

Первый раз:

1. Создай репозиторий на GitHub (кнопка **New**). Можно приватный —
   Pages работает и с приватными на платном плане, на бесплатном нужен публичный.
2. Залей туда содержимое этой папки. Если через сайт GitHub — просто перетащи
   файлы в окно браузера (**Add file → Upload files**).
3. В репозитории: **Settings → Pages → Source: Deploy from a branch**,
   ветка `main`, папка `/ (root)` → **Save**.
4. Через пару минут сайт будет по адресу `https://ИМЯ.github.io/РЕПОЗИТОРИЙ/`.

### Свой домен

1. **Settings → Pages → Custom domain** → впиши свой домен → **Save**.
   GitHub сам создаст файл `CNAME` в репозитории — не удаляй его.
2. У регистратора домена пропиши DNS-записи:
   - для домена вида `example.com` — четыре A-записи на
     `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - для поддомена вида `www.example.com` — CNAME-запись на `ИМЯ.github.io`
3. Когда DNS обновится (от 10 минут до суток), включи **Enforce HTTPS**.

### Важные детали

- **`.nojekyll`** — пустой файл в корне. Без него GitHub прогоняет сайт через
  Jekyll, который игнорирует папки, начинающиеся с `_` и `.`. Не удаляй.
- **Регистр букв важен.** На GitHub `Styles.css` и `styles.css` — разные файлы,
  на Windows — один и тот же. Поэтому ссылка с опечаткой в регистре работает
  локально, но ломается на сайте. Все ссылки в проекте проверены.
- **Все пути относительные** (кроме `404.html`), поэтому сайт работает и в
  подпапке, и на своём домене.
